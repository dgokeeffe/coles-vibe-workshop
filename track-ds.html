Lab 1 / Phase 2: Window Functions Reference

Data Science

Lag Features (PySpark Window)

from pyspark.sql import Window from pyspark.sql import functions as F window = Window.partitionBy("state", "industry") \ .orderBy("month") df = df.withColumn( "turnover_lag_1m", F.lag("turnover_millions", 1).over(window) ).withColumn( "turnover_lag_3m", F.lag("turnover_millions", 3).over(window) ).withColumn( "turnover_lag_12m", F.lag("turnover_millions", 12).over(window) )

Seasonal Features

df = df.withColumn( "month_of_year", F.month("month") ).withColumn( "quarter", F.quarter("month") ).withColumn( "is_december", F.month("month") == 12 ).withColumn( "is_q4", F.quarter("month") == 4 )

Growth Features

# Month-over-month growth df = df.withColumn( "turnover_mom_growth", (F.col("turnover_millions") - F.lag("turnover_millions", 1) .over(window)) / F.lag("turnover_millions", 1) .over(window) * 100 ) # Year-over-year growth df = df.withColumn( "turnover_yoy_growth", (F.col("turnover_millions") - F.lag("turnover_millions", 12) .over(window)) / F.lag("turnover_millions", 12) .over(window) * 100 )
Common mistake: The agent may try to use pandas. If it does, say: "Use PySpark Window functions, not pandas. Check CLAUDE.md."

2.1 Register with Alias-First Pattern

Paste into Claude Code
import mlflow # DON'T SKIP: point the registry at UC mlflow.set_registry_uri("databricks-uc") # Find best run (highest R²) from Lab 1 # Register + set @challenger alias result = mlflow.register_model( model_uri=f"runs:/{best_run_id}/model", name="workshop_vibe_coding.TEAM_SCHEMA." "grocery_forecaster", ) client = mlflow.MlflowClient() client.set_registered_model_alias( name="workshop_vibe_coding.TEAM_SCHEMA." "grocery_forecaster", alias="challenger", version=result.version, )
Don't skip this: mlflow.set_registry_uri("databricks-uc") must run at session start — without it, MLflow writes to the workspace registry, not UC. The agent will silently fail this unless CLAUDE.md reminds it.

Aliases, Not Version Numbers

Use @champion for the current best model and @challenger for a new version under review. Promote @challenger → @champion once you've validated it.

# Promote challenger to champion client.set_registered_model_alias( name="workshop_vibe_coding.TEAM_SCHEMA." "grocery_forecaster", alias="champion", version=result.version, )
Why aliases: Downstream code references @champion, not version 7. You can swap the underlying version without touching the inference code.

UC + MLflow: The Six Rules Your Pair Should Pin

Data Science
Phase 2 • Reference

These six rules cover 95% of the friction you'll hit between MLflow and Unity Catalog. Add them to your CLAUDE.md before Phase 2 so the agent follows them by default.

Airgap-safe deep-dive: the full MLflow + UC pattern library, troubleshooting cheat sheet, and "everything is on fire, just need to demo" fallback is in starter-kit/references/mlflow-uc.md. Agent-readable. @reference sections in your prompts (e.g. “Read @starter-kit/references/mlflow-uc.md §3, then register…”).

1. Three-level names, always

Never model_name alone. Always catalog.schema.model_name — e.g. workshop_vibe_coding.TEAM_SCHEMA.grocery_forecaster.

2. Set the registry URI

mlflow.set_registry_uri("databricks-uc") at session start. Without it, models silently land in the legacy workspace registry.

3. Aliases > version numbers

Reference @champion and @challenger in code. Swap the underlying version by moving the alias — not by editing downstream code.

4. Never ai_query on a custom UC model

ai_query requires a serving endpoint. For batch inference, use mlflow.pyfunc.load_model (notebook) or mlflow.spark_udf (pipeline).

5. Verify with DESCRIBE MODEL

Navigator's V-step: DESCRIBE MODEL workshop_vibe_coding.TEAM_SCHEMA.grocery_forecaster — or open the model page in the UC UI. Confirm version + alias before scoring.

6. Checkpoint fallback

If registration breaks, use workshop_vibe_coding.checkpoints.grocery_forecaster@fallback. Pre-registered, always available, same feature schema.

Lab 2 / Phase 3: Tier 2 Stretch — Lakeflow Batch Job

Data Science
Phase 3 • StretchOnly if 20+ min left
⚠ DO NOT use ai_query. ai_query requires a live serving endpoint, which we are deliberately avoiding in this workshop. Use mlflow.spark_udf to load the UC model directly into the pipeline — it distributes across the cluster and doesn't need an endpoint.

Tier 2 productionizes the batch inference: instead of re-running the notebook manually, a scheduled Lakeflow job refreshes gold_forecast on a cadence you set.

Driver (keyboard)

Add a new @dp.materialized_view to your Lab 1 pipeline (or a fresh one). Load the model with mlflow.spark_udf, apply it as a column expression, write to gold_forecast.

Navigator (reads + verifies)

Verify the materialized view lands with the expected row count and a predicted_turnover column. Open Genie and ask "what's the forecast turnover for Victoria next month?" — that's your demo query.

Paste into Claude Code
import databricks.declarative_pipelines as dp import mlflow from pyspark.sql import functions as F mlflow.set_registry_uri("databricks-uc") # spark_udf loads the UC model once per executor # No serving endpoint needed. forecast_udf = mlflow.pyfunc.spark_udf( spark, model_uri="models:/workshop_vibe_coding.TEAM_SCHEMA" ".grocery_forecaster@champion", result_type="double", ) @dp.materialized_view( name="gold_forecast", comment="Monthly turnover forecast by state/industry", ) def gold_forecast(): features = dp.read("retail_features") return features.withColumn( "predicted_turnover", forecast_udf(*[F.col(c) for c in feature_cols]), )
Why spark_udf (not ai_query): spark_udf loads the pyfunc model directly onto each executor — zero endpoint provisioning time, no per-request latency, distributes naturally. ai_query is for Foundation Model endpoints (Claude, Llama), not your custom UC model.
Stretch only. If you're still debugging the notebook at 20 min remaining — skip Tier 2. A clean Tier 1 demo beats a half-finished Tier 2 pipeline.
